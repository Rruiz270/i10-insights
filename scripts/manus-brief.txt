# WRITING RULES

- Write each language NATIVELY — do NOT translate one to the other.
- Length: 600-900 words per language.
- Structure: lede paragraph (the story in 2 sentences) → context (why this matters) → BR angle → i10 perspective → close with open question.
- Use ## markdown headers for sections (3-4 sections max).
- USE blockquote markdown (`> `) for at least ONE pulled-out sentence per language — pick a sentence that distills the article's claim. The template renders the FIRST `> ` as a "Key insight" callout at the top, AND renders all `> ` blocks as styled pull-quotes inline.
- USE inline links: `[descriptive text](https://full-url)`.
- Use **bold** sparingly — only for key statistics like **34%**. Stats in bold get rendered with green background highlight by the template.
- Use ordered lists `1. ... 2. ...` when listing 3+ enumerated items — they render as numbered cards with cyan badges.
- Use bullet lists `- ` for short non-enumerated items.
- Quotes: max 25 words, with attribution.
- Headlines: 50-70 chars, descriptive not clickbait.
- No emoji in body. No exclamation marks except in direct quotes.

- Include alt text in the SAME LANGUAGE as the body it appears in (PT for body_pt, EN for body_en — this means generating 2-3 images for PT AND 2-3 for EN, OR using the same image URLs with different alt text in both bodies).

# HERO IMAGE — strict direction

Pick ONE of these two valid styles based on what fits the article's topic:

## Style A — Documentary photograph (preferred for politica, sala_de_aula, sometimes pesquisa)

A real-looking, candid photograph of Brazilian public education context.

GOOD subjects:

- A teacher engaging with students, with technology integrated naturally

## Style B — Editorial illustration (preferred for ferramentas, pesquisa-with-data, etica)

A geometric/abstract illustration in i10's brand palette.

Required elements:
